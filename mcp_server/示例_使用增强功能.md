> "爬取 https://example.com 失败了，怎么回事？"

工具: diagnose_crawl_failure

📄 HTML分析:
  • 文档长度: 2456 字符
  • 标题: Example Domain
  ⚠️ 检测到SPA特征（#app/#root/vite），需要JS渲染
     - #app存在: True

## 场景6：复杂需求 - 组合使用工具

### 用户意图
> "帮我爬取这三个网页，如果哪个质量不好就重写，
> 最后生成一份总结报告"

### AI操作流程

**第1步：逐个爬取并评估**
```
对于每个URL:
  工具: crawl_with_quality_check
  参数:
    url: "..."
    auto_rewrite: true  # 自动重写低质量内容
    llm_provider: "deepseek"
```

**第2步：收集所有生成的文件路径**

假设生成了：
- `/path/to/file1_regenerated.md` (重写后)
- `/path/to/file2.md` (原始，质量好)
- `/path/to/file3_regenerated.md` (重写后)

**第3步：用大模型生成总结**

读取所有文件内容，然后：

```
工具: crawl_and_regenerate_with_llm
参数:
  input_file: "/path/to/merged_content.md"  # 合并后的内容
  provider: "bailian"
  model_name: "qwen-plus"
  prompt_template: """
这是三个网页的内容，请生成一份结构化的总结报告：

1. 概述每个网页的主要内容
2. 提取共同的主题和要点
3. 做横向对比分析
4. 生成结论和建议

内容：
{content}
  """
```

## 常用参数组合

### 爬取普通网页（质量未知）
```python
crawl_with_quality_check(
    url="...",
    output_dir="/path/to/output",
    auto_rewrite=False,  # 先评估，不自动重写
    verbose=True
)
```

### 爬取已知低质量网页（自动重写）
```python
crawl_with_quality_check(
    url="...",
    output_dir="/path/to/output",
    auto_rewrite=True,  # 自动重写
    llm_provider="deepseek",
    verbose=True
)
```

### 批量爬取+AI总结
```python
crawl_and_summarize(
    urls=["url1", "url2", "url3"],
    output_dir="/path/to/output",
    llm_provider="bailian",
    llm_model="qwen-plus",
    save_originals=True  # 保存原始文件
)
```

### 诊断爬取问题
```python
diagnose_crawl_failure(
    url="...",
    verbose=True  # 详细日志
)
```

### 重写已有文件
```python
crawl_and_regenerate_with_llm(
    input_file="/path/to/file.md",
    provider="deepseek",
    save_original=True  # 保留原文件
)
```

## 最佳实践建议

### 1. 首次爬取未知网站
```
第1步: crawl_with_quality_check(url, auto_rewrite=False)
第2步: 根据质量报告决定是否重写
```

### 2. 已知网站质量不好
```
直接使用: crawl_with_quality_check(url, auto_rewrite=True)
```

### 3. 爬取失败
```
第1步: diagnose_crawl_failure(url)
第2步: 根据诊断结果调整参数重新爬取
```

### 4. 批量处理
```
使用: crawl_and_summarize(urls, llm_provider, llm_model)
一步完成爬取+总结
```

### 5. 避免重复爬取
```
如果已有文件: crawl_and_regenerate_with_llm(input_file=...)
而不是: crawl_and_regenerate_with_llm(url=...)
```

## 总结

新的增强工具让爬取工作变得：

1. **更智能** - 自动评估质量，自动决定是否需要重写
2. **更快速** - 诊断工具快速定位问题，避免反复试错
3. **更灵活** - 支持单个/批量、爬取/重写、自动/手动等多种模式
4. **更高效** - 批量总结工具一步完成爬取+AI分析

选择合适的工具，让AI助手更好地理解和执行你的意图！
