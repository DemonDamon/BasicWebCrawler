**用例总览**
- 覆盖简单和复杂两类场景，展示“用户指令 → 所触发工具/函数 → 返回结果”的完整链路
- 所有工具均由 MCP 服务器提供，核心实现位于 `mcp_server/mcp_server.py` 和 `crawler.py`
- 如需流式网络通信，使用 `streamableHttp` 或 `SSE` 传输启动服务器

**简单场景**
- 单页抓取为 Markdown（默认）
  - 用户指令：`“爬取 https://example.com 并生成 Markdown”`
  - 触发工具：`crawl_single_url`
  - 关键调用：`crawler.py:557` 的 `fetch_with_retry` → `crawler.py:636` 的 `fetch_and_convert_to_markdown`
  - 结果：返回成功信息和保存路径，图片默认下载到本地
  - 入口：`mcp_server/mcp_server.py:59`
- 输出纯文本（不含 Markdown）
  - 用户指令：`“交互式抓取 https://docs.example.com 输出纯文本”`
  - 触发工具：`interactive_crawl`，参数：`{"return_format":"text","actions":[]}`
  - 关键调用：`crawler.py:879` 的 `convert_html_to_text`
  - 结果：返回结构化 JSON，`format="text"`，`content` 为纯文本
  - 入口：`mcp_server/mcp_server.py:457`
- 关闭 GFM 特性（输出 CommonMark 风格）
  - 用户指令：`“交互式抓取 https://blog.example.com，关闭 GFM”`
  - 触发工具：`interactive_crawl`，参数：`{"no_gfm":true}`
  - 关键调用：`crawler.py:831` 的 `apply_no_gfm` 在 `convert_html_to_markdown` 之后处理
  - 结果：表格、任务列表、删除线、角括号自动链接被剥离
  - 入口：`crawler.py:838`
- 保留远程图片链接不下载
  - 用户指令：`“交互式抓取 https://news.example.com，图片不要下载，保持远程链接”`
  - 触发工具：`interactive_crawl`，参数：`{"retain_images":true}`
  - 关键调用：`crawler.py:838` 的 `convert_html_to_markdown` 按参数跳过下载
  - 结果：Markdown 中图片链接指向原站
- 保留 `data:` URL 图片
  - 用户指令：`“交互式抓取 https://graphics.example.com，保留 data URL 图片”`
  - 触发工具：`interactive_crawl`，参数：`{"keep_img_data_url":true}`
  - 关键调用：`crawler.py:838` 的 `convert_html_to_markdown` 在图片处理环节保留 `data:` 链接
- 生成图片/链接摘要
  - 用户指令：`“交互式抓取 https://portal.example.com，附带图片摘要和链接摘要”`
  - 触发工具：`interactive_crawl`，参数：`{"with_images_summary":true,"with_links_summary":true}`
  - 关键调用：`crawler.py:1122` 的 `images_summary`、`crawler.py:1135` 的 `links_summary`
  - 结果：结构化 JSON 的 `summaries.images`/`summaries.links` 填充统计数据
- 禁用缓存/启用缓存
  - 用户指令：`“交互式抓取 https://docs.example.com，禁用缓存”`
  - 触发工具：`interactive_crawl`，参数：`{"no_cache":true}`
  - 关键调用：`crawler.py:1167` 的 `DiskCache`（键生成与读写）；禁用时跳过 `get/set`
  - 结果：每次均重新渲染获取 HTML

**交互式场景**
- 登录后抓取（输入账号密码、提交表单）
  - 用户指令：`“交互式抓取 https://account.example.com，先登录，再抓取主页面内容输出 Markdown”`
  - 触发工具：`interactive_crawl`
  - 参数示例：
    ```json
    {
      "url": "https://account.example.com",
      "actions": [
        {"type":"wait","selectors":["#login-form"],"timeoutMs":15000},
        {"type":"type","selector":"#username","text":"user@example.com"},
        {"type":"type","selector":"#password","text":"pa55word","submit":true},
        {"type":"wait","selectors":["#dashboard"],"timeoutMs":15000}
      ],
      "return_format": "markdown",
      "retain_images": false,
      "with_links_summary": true
    }
    ```
  - 关键调用：Playwright 交互流程在 `crawler.py:906` 同步实现与 `crawler.py:989` 异步实现；封装线程执行 `crawler.py:1094`
  - 结果：返回结构化 JSON，包含 `content`、`metadata/external`、`summaries.links`
- 展开目录后抓取（点击、滚动、等待）
  - 用户指令：`“交互式抓取 https://docs.example.com，点击展开目录，滚动到底再抓取为纯文本”`
  - 触发工具：`interactive_crawl`
  - 参数示例：
    ```json
    {
      "url": "https://docs.example.com",
      "actions": [
        {"type":"wait","selectors":["#toc"],"timeoutMs":10000},
        {"type":"click","selector":"#toc .expand-all"},
        {"type":"scroll","to":"bottom"},
        {"type":"sleep","ms":800}
      ],
      "return_format": "text",
      "no_gfm": true,
      "with_links_summary": true
    }
    ```
  - 关键调用：`crawler.py:879` 的 `convert_html_to_text`；链接汇总 `crawler.py:1135`

**批量与提取**
- 从文本中提取 URL 并批量爬取合并输出

  - 触发工具：`crawl_urls_from_text`
  - 关键调用：`crawler.py:603` 的 `extract_urls_from_text` → `crawler.py:1191` 的 `process_url_text_mode`
  - 结果：生成一个合并后的 `.md` 文件，包含统计摘要与图片目录
  - 入口：`mcp_server/mcp_server.py:205`
- 仅提取 URL 列表并标注是否需要 Cookies
  - 用户指令：`“从这段文字里提取所有 URL 并标注是否需要 Cookies”`

  - 结果：返回 URL 列表与站点配置提示
  - 入口：`mcp_server/mcp_server.py:289`

**生成与重写**

**流式 HTTP / SSE**
- 启用 HTTP 流式传输（建议用于远程/云端）
  - 命令：`python mcp_server/start_mcp_server.py --transport http --host 0.0.0.0 --port 9000 --auto`
  - 客户端配置：
    ```json
    {
      "mcpServers": {
        "basic-web-crawler": {
          "type": "streamableHttp",
          "url": "http://localhost:9000/mcp"
        }
      }
    }
    ```
  - 入口：启动调用 `mcp.run(transport="http", host=..., port=..., path="/mcp")` 于 `mcp_server/start_mcp_server.py:147`
- 启用 SSE（适合 Web 应用的服务器推送）
  - 命令：`python mcp_server/start_mcp_server.py --transport sse --host 127.0.0.1 --port 8000 --auto`
  - 客户端配置：
    ```json
    {
      "type": "sse",
      "url": "http://127.0.0.1:8000/sse"
    }
    ```
  - 入口：`mcp_server/start_mcp_server.py:146`

**触发逻辑概览**
- `crawl_single_url` → `fetch_with_retry` → `fetch_and_convert_to_markdown` → 保存 `.md`（`mcp_server/mcp_server.py:59`，`crawler.py:557/636`）
- `interactive_crawl` → Playwright 交互渲染 → `convert_html_to_markdown` 或 `convert_html_to_text` → 结构化 JSON（`mcp_server/mcp_server.py:457`，`crawler.py:1094/838/879`）
- `crawl_urls_from_text` → `extract_urls_from_text` → 批量抓取合并（`mcp_server/mcp_server.py:205`，`crawler.py:603/1191`）
- `extract_urls` → 纯 URL 列表 + 配置（`mcp_server/mcp_server.py:289`，`crawler.py:603/515`）
- `check_site_config/get_supported_sites` → 站点配置展示（`mcp_server/mcp_server.py:321/350`，`crawler.py:515`）
- `crawl_and_regenerate_with_llm` → 抓取 + LLM 重写（`mcp_server/mcp_server.py:578`，`mcp_server/llm_client.py:75`）
