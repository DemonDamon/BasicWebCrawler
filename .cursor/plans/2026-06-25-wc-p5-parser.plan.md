## 目标
从微信公众号文章 DOM 稳定提取结构化字段，选择器配置化，页面变更时改配置不改代码。

## 复用现有能力
- `crawler.py` 的 HTML→Markdown、图片本地化、`#js_content` 主体抽取逻辑抽成可调用函数。
- Playwright 渲染路径复用于「需要 JS 的快照回放」。

## 交付物
- `wechat_collector/parsers/wechat.py`：输入 HTML（或 URL+HTML），输出方案 §4.4 字段：
  title / account_name / wechat_id / biz / url / canonical_url / publish_time / cover_url / summary / content_html / content_text / collected_at。
- `wechat_collector/parsers/wechat.json`：配置化选择器（方案 §4.6）：
  - title: `meta[og:title]`, `#activity-name`
  - account_name: `#js_name`, `.profile_nickname`
  - content: `#js_content`
  - cover: `meta[og:image]`
  - publish_time: `#publish_time`, `meta[property=...]`
- 失败处理：解析失败保存 HTML 快照到 `samples/snapshots/` + 失败原因。
- 回归测试：`tests/test_wechat_parser.py` 基于固定 HTML 样本断言字段。

## 设计要点（方案 §4.6 / §7.2）
1. 选择器按优先级列表逐个尝试，命中即止。
2. 建样本文章库用于页面变更后回归。
3. content_html 清洗（去脚本/无关节点）后再算 content_text 与 content_hash。

## 验收

- [x] 删除主选择器后仍能用 fallback 选择器命中
- [x] `pytest tests/test_wechat_parser.py` 通过
